# gstack Version — Magic Brooms

This is the gstack version of Magic Brooms, using Garry Tan's virtual engineering team framework.

## Devlog Rule

After completing any major milestone (office hours, planning, design, implementation, QA, review, etc.), **automatically** create a devlog entry in `../devlog/` and update `../devlog/README.md`. Use the prefix `gstack-NNN` (e.g., `gstack-001-office-hours.md`). Check existing `gstack-*` entries in `../devlog/` first to pick the next number. See the root `CLAUDE.md` for content details.
