We're building a Zulip-like web chat application called "Magic Broom Chat" for an educational course on AI-assisted software development.

**Core concept:** A real-time team chat app inspired by Zulip, starting with a straightforward channel-based messaging model. Topic-based threading (Zulip's key differentiator) is a future goal but not part of the MVP.

**MVP features:**
- User authentication (sign up, log in, log out)
- Channels (create, join, browse)
- Real-time messaging within channels
- User presence (online/offline/idle)
- Basic search across messages

**Constraints:**
- Start simple and iterate — get something working end-to-end first
- The tech stack is your choice — pick what best fits this type of app
- This will be a web app (no native mobile needed)
- Prioritize a clean, functional UI over visual polish
