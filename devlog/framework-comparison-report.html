Why gstack Wins

It's not about features — it's about feel.

All five implementations have the same core features: auth, channels, messaging, search, presence. The difference is in the details that make people want to come back:

  • Temporal UX: Animations and transitions make state changes feel intentional, not jarring. Messages slide in, channels cross-fade, typing dots bounce.
  • Message grouping: Consecutive messages from the same user collapse into a compact stream — like Slack, not like a forum.
  • Optimistic updates with retry: Messages appear instantly. If the network fails, a red border and retry button appear inline. No lost messages.
  • Cmd+K command palette: Power user navigation. Jump to any channel without scrolling the sidebar.
  • Skeleton loaders: Loading states show the shape of what's coming, not a blank screen or spinner.
  • The RLS fix: gstack discovered and fixed an infinite recursion bug in Supabase RLS policies that none of the other four builds caught.

Cross-Framework Learnings

Patterns discovered by building the same app five different ways:

1

Broadcast Before Send

Subscribe to Supabase Broadcast channels before sending. Broadcasts silently fail if the recipient isn't subscribed yet.

2

PostgREST FK Hints

Messages→profiles joins need explicit FK hints: profiles!messages_user_id_fkey(*). PostgREST can't resolve ambiguous FKs automatically.

3

Presence Cleanup is Unreliable

Don't write "offline" status in React cleanup functions. Let Supabase Presence handle ephemeral state automatically.

4

RLS Recursion Trap

RLS policies that cross-reference each other (channels ↔ channel_members) cause infinite recursion and 500 errors. Simplify to one-directional checks.

5

Schema-First Pays Off

Design your database before your UI. Generated TypeScript types from a live schema prevent type mismatches and catch issues early.

6

AI Uses Stale Patterns

Models default to older, better-documented APIs (like postgres_changes) over newer ones (Broadcast + DB Triggers). Always verify against current docs.

7

Worktree Isolation Matters

Without git worktrees, parallel AI sessions contaminate each other's branches. Vanilla's direct-to-main approach created git status pollution for everyone.
